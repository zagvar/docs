## Two independent foundations

Mosaic and Relay are designed to work well in the same trading platform, but
neither package depends on the other.

- **Mosaic** owns reusable trading UI behavior, accessible interaction, frontend
  validation, and presentation-ready domain state.
- **Relay** owns normalized market-data contracts, provider integration seams,
  hot caching, pub/sub, hydration, and realtime transport.
- **Your application** owns the mapping between them, along with authentication,
  accounts, balances, order execution, permissions, compliance, and product
  policy.

This separation lets developers adopt either project independently and prevents
a provider, broker, framework, or product model from becoming a hidden system
requirement.
