## Requirements

- React 18.3 or React 19
- a React application with its own styling system
- pnpm 11 when working in the Mosaic repository

## Explore the repository

```bash
git clone https://github.com/zagvar/mosaic.git
cd mosaic
pnpm install
pnpm dev
```

The demo includes an equity trade ticket, a crypto order book, mocked HTTP and
WebSocket market data, selectable quote and book prices, and order review.

## Choose a package

Use `@zagvar/mosaic-core` when you need framework-independent trading contracts,
schemas, validation, summaries, order state reducers, or order-book
reconciliation.

Use `@zagvar/mosaic-react` when you need accessible components and React state
hooks. It already depends on Mosaic Core.

Install only the packages your application imports directly:

```bash
pnpm add @zagvar/mosaic-react
pnpm add @zagvar/mosaic-core
pnpm add @zagvar/decimal
```

The last two commands are optional. Add Mosaic Core when application code uses
its contracts or framework-free utilities. Add Decimal when application code
validates, normalizes, compares, or calculates exact decimal values. Import
Decimal APIs from `@zagvar/decimal`, not through Mosaic.

## Application responsibilities

Before composing components, identify where your application will provide:

- normalized instrument and market-data state
- asset-specific order rules
- account balances and buying power
- server-authoritative order previews
- order submission and lifecycle state
- localized messages and application styling

Mosaic intentionally receives these concerns through props and application
state rather than choosing providers or backend architecture for you.
