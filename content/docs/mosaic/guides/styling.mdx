---
title: Styling
description: Apply a product design system through explicit Mosaic class-name slots.
---

Mosaic components ship behavior and semantic structure rather than a branded
theme. Each component accepts a typed `classNames` object.

```tsx
<QuoteDisplay
  quote={quote}
  quoteCurrency="USD"
  classNames={{
    root: "quote-card",
    quotes: "quote-grid",
    quote: "quote-cell",
    price: "quote-price",
    spread: "quote-spread",
  }}
/>
```

Slots can contain plain CSS classes, CSS Module values, Tailwind utilities, or
classes generated by another styling system.

## Recommended approach

1. Style the root layout and component spacing.
2. Style semantic states such as bid, ask, warning, disabled, and pending.
3. Verify focus visibility and keyboard interaction.
4. Test long translations and large formatted numbers.
5. Test narrow containers rather than assuming a full-page layout.

Do not remove focus outlines without an accessible replacement. Do not use color
as the only distinction between buy and sell, bid and ask, or success and error.

Mosaic's markup may evolve before stable release. Treat documented class-name
slots—not descendant selectors tied to internal DOM shape—as the styling API.
