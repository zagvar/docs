## Requirements

- Node.js 22 or later
- an ESM TypeScript or JavaScript project
- Redis only when using `@zagvar/relay-redis`

## Explore the repository

```bash
git clone https://github.com/zagvar/relay.git
cd relay
pnpm install
pnpm check
pnpm test:run
pnpm build
```

## Choose the packages

Every Relay service begins with the core package. Once the packages are
published, installation will use the standard workspace commands:

```bash
pnpm add @zagvar/relay-core
```

Add infrastructure packages as required:

```bash
pnpm add @zagvar/relay-redis redis
pnpm add @zagvar/relay-ws ws
pnpm add @zagvar/relay-adapter-alpaca
```

You do not need to install every package. A service can use the core contracts
with its own cache, event bus, transport, and provider adapter.

These commands are shown for the upcoming package release and are not expected
to resolve from npm yet.

## Minimal in-memory pipeline

```ts
import {
  MarketDataPipeline,
  MemoryMarketDataCache,
  MemoryRelayEventBus,
} from "@zagvar/relay-core";

const cache = new MemoryMarketDataCache();
const eventBus = new MemoryRelayEventBus();
const pipeline = new MarketDataPipeline({ cache, eventBus });

await pipeline.processEvent({
  type: "trade",
  symbol: "BTC/USD",
  assetClass: "crypto",
  venue: "COINBASE",
  price: 68_250.5,
  quantity: 0.02,
  timestamp: "2026-07-12T01:30:00.000Z",
});
```

The in-memory implementations are useful for tests, examples, and
single-process demos. Production services can replace them with Redis or custom
implementations of the same core interfaces.

## Package boundaries

- Import shared contracts and processing primitives from `@zagvar/relay-core`.
- Use `@zagvar/relay-redis` when state or pub/sub must cross processes.
- Use `@zagvar/relay-ws` to hydrate and stream normalized data to clients.
- Keep provider credentials and raw payload mapping inside a provider adapter.

Relay packages are intentionally composable rather than bundled into a required
server framework.
